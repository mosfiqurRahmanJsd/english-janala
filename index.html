Frequently Asked Questions

1. What is the difference between var, let and const ?
Ths main difference between var, let and const is there scope and reassignment/redeclaration rules. var is function scope and hoisted, while let and const have block scope and are not hoisted to the same extent. const variable cannot be reassigned, whereas let can be reassigned but not redeclared, and var can be both reassigned and redeclared.
2. What is the difference between map(), forEach() and filter() ?
map(), forEach(), and filter() are array iteration methods in JavaScript, each serving a distinct purpose: forEach(): Purpose: To execute a provided function once for each array element. Return Value: undefined. It does not create a new array or modify the original array unless explicitly done within the callback function. Use Case: When you need to perform an action for each element, such as logging values, updating a database, or triggering side effects, without needing a new array as a result. map(): Purpose: To create a new array by applying a provided function to each element of the original array. Return Value: A new array of the same length as the original, containing the results of calling the provided function on every element. The original array remains unchanged. Use Case: When you need to transform each element of an array into a new value and collect these transformed values into a new array.filter(): Purpose: To create a new array containing only the elements that satisfy a provided test function. Return Value: A new array containing only the elements for which the provided callback function returns true. The original array remains unchanged. Use Case: When you need to select a subset of elements from an array based on a specific condition.Use forEach() for side effects or when you don't need a new array. Use map() when you need to transform each element and create a new array of transformed values. Use filter() when you need to select elements based on a condition and create a new array containing only those selected elements.
3. What are arrow functions, and how are the different from regular functions?
Arrow functions are a more concise syntax for writing functions introduced in ES6 that differ from regular functions primarily in their handling of this, arguments, and lack of their own prototype. Arrow functions use lexical this, meaning they inherit this from the surrounding code, whereas a regular function's this is determined by how it is called. They also don't have their own arguments object, no prototype property, and cannot be used as constructors.
4. How do JavaScript promise work?
JavaScript Promises provide a structured way to handle asynchronous operations, representing the eventual completion (or failure) of an operation and its resulting value. They address the challenges of callback-based asynchronous code, such as "callback hell."
5. How does closer work in JavaScript?
In JavaScript, a closure is the combination of a function and its surrounding state (its lexical environment). Closures allow an inner function to retain access to variables from its outer scope, even after the outer function has finished executing.
